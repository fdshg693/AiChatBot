## 現在ある機能

### チャット
- メッセージを送信し、AI から**ストリーミング（逐次表示）**で返答を受け取れる。
- AI は汎用アシスタント（特定ドメインに限定しない、フレンドリーな受け答え）。
- 返答の生成中は送信ボタンが無効化される（二重送信の防止）。
- 会話が空のときは案内文（プレースホルダ）を表示する。

### ツール呼び出し（Web 検索・本文抽出）
- AI が必要に応じて**外部ツールを呼び出す**（多段：ツール呼び出し→結果→応答、最大 5 ステップ）。
- 提供ツールは **Web 検索（tavilySearch）** と **本文抽出（tavilyExtract）**（Tavily）。
- どのツールを使うかは**画面のトグルで都度選択**でき、会話の途中でも切り替えられる。
  - 選択はリクエストごとの状態で、会話履歴には残さない。
  - フロントが何を送っても、バックエンドの許可リストに無いツールは有効化されない。

### 会話の保持
- やり取りしたメッセージ（ユーザー発言＋AI 応答）が**サーバ側に永続化**される。
- 会話には**タイトルが自動付与**される（最初のユーザー発言の先頭から生成）。

### 会話の切り替え（UI 対応済み）
- サイドバーに過去会話が**更新の新しい順**で並ぶ。
- 過去会話を選ぶと**履歴を読み込んで続きから**やり取りできる。
- 「新規会話」で別スレッドを開始できる。

### 会話の参照（バックエンド API として提供）
- 会話の**一覧**を取得できる（更新が新しい順）。
- 指定した会話の**メッセージ履歴**を取得できる。

## まだ無い機能（現状の制限）

- **認証・ユーザー管理が無い**：誰でも同じデータにアクセスする前提。
- **添付ファイル／画像入力が無い**：テキストメッセージのみ。
- **メッセージの編集・再生成・削除が無い**。
- **会話の改題・削除が無い**（一覧表示・切り替えはできるが、リネーム/削除の操作は未対応）。
- **RAG（社内文書などの検索回答）は無い**。

## 利用上のメモ
- 既定の応答モデルは環境変数で差し替え可能（モデル指定は運用設定の領域）。
- Web 検索／本文抽出を使うには `TAVILY_API_KEY`、AI 応答には `ANTHROPIC_API_KEY` が必要。
  - 起動時に未設定なら警告ログを出す。ツール呼び出しの内容やストリームのエラーもログに残る
    （詳細表示は `LOG_LEVEL=debug`）。ログは画面と `backend/logs/app.log` の両方へ出力し、
    サイズ超過で自動ローテーションする。
- 開発時はフロント（Vite）とバックエンドを同時に起動して使う。
